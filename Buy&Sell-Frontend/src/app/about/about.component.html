OBJECTIVES

Providing a platform to perform shopping

Ecommerce hosts a platform that lets businesses sell their products or services throughout the world through an electronic medium. It includes all sorts of businesses categories that used by the island wide service & goods suppliers and the general public

Providing secured platform

For as long as there has been email, businesses have relied on verifying email addresses to prove ownership of newly created accounts. But in today’s climate, assur-ing new user signups are real human beings is becom-ing critical. By using the phone number verifica-tion,(OTP codes) the users can experience a stronger level of security and it is gradually becoming the pre-ferred way to validate new account ownership.

Reducing management cost

Increasing the efficiency of the services

With the evolving technology online ecommerce plat-form can provide efficient way for shopping for both customers and the sellers.

Providing a feature for communication

Integrating effective communication techniques will help develop strong supplier relationships by promot-ing trust, improving risk management, and supporting long-term goals

Machine learning approach to introduce image-based search mechanism

Also known as the visual searcg and it can be used to helpthrough images even though the users are short of verbal information to put into words. This helps the users to search for their desired product and also opens all the similar products that might like.

Making responsive ecommerce website

With the increasing use of smartphones for shopping online, it has become more than mandatory for ecom-merce businesses to go mobile. By responsive, it means to create a website that can be viewed from any devices of varying screen size, equally. Most commonly used Google search engine is already ranking the web pages according to the responsiveness of the websites. Making the websites responsive lets designers and users to op-timize the features

Method of implemenatation

Step 01 Requirement specification

Identifying the requirements and creating the priority list to focus on each segment and functionality of the project.

Step 02 Creating wireframes and prototype

Using the wireframes brings more clarity to the project and it helps to visualize the views and identify the goals achieved by each segment. Using prototypes give the overview of the finalized product.

Step 03 Designing the architecture

Creating a logical layout of the website according to the requirements gathered.

Step 04 Deciding the development method

A website can be developed using many approaches such as implementing all the logical fuctionalities per-formed in the frontend, using a frontend with backend as a service model such as firebase, using a front end with along with a backend. Front end with a backend is the development method used for the implementation of the project.

Step 05 Selection of framworks to implement the frontend and the backend

Step 06 Selection of the database model

Relational, non relational, Object oriented was heavily used database models and Non relational noSQL ap-proach was more appropriate for the application be-cause of it can handle large volumes of data at high speed and it is more developer friendly option.

Step 07 Implementation the features

Implementing each feature such as Signup/Login, mo-bile authentication, messaging platform, advertisement posting section, image identification process using google compute engine.

Step 08 Testing and debugging

Unit testing, integration testing, System testing was per-formed at each stage and followed CICD practice while developing the project

Step 09 Deployment

Front end and back end was hosted in AWS but later it had to be removed because of the expiration of the ser-vices.

Challenges

Choosing the appropriate language and framework for the development process

Cross Origin Errors

A security mechanism that browsers implement called the same-origin policy. The same-origin policy fights one of the most common cyber attacks out there: cross-site request forgery. In this maneuver, a malicious web-site attempts to take advantage of the browser's cookie storage system. Using access control allow origin meth-od this error can be rectified. Adding appropriate head-ers from the servers is another method of solving this issue.

Frequent updates in web frameworks

In order to prevent the security vulnerabilities both front end and backend frameworks releases the updates to the softwares. This was a problem which couldn’t be solved using any methods other than the re-writing the program. In the beginning of the project the stable ver-sion of angular was angular 8 and now it is angular 10 also some services used in the firebase is not compatible with older vesions of the angular.

Integration of the machine learning models with the front end

Less documentation for the integrating a machine learning model was another challenged faced during the im-plementation of the project.
